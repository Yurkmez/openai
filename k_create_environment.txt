Библиотека openai находится по адресу
https://pypi.org/project/openai/

Чтобы устанавливать ее не глобально
pip install openai
а в виртуальной среде, то следует ее создать. 

1. ______________
Для этого, на том же ресурсе есть программа  - pipenv
которая упрощает процесс создания виртуальной среды.
Его как раз и надо установть глобально, чтобы упрощенно 
создавать под каждый проект свою виртуальную среду
Устанавливаем:

pip install pipenv

2. ________________ 
Затем желательно создать отдельную папку (в папке python)
.venv
где файлы по виртуальной среде и будут  находиться.

3. _________________
Переходим в папку python

cd python

и там устанавливаем openai

pipenv install openai

Мы видим 
(а)
Successfully created virtual environment!
Virtualenv location: C:\Users\shaps\Desktop\Machine_learning\Bogdan\python\.venv   
что по такому пути создана виртуальная среда
(б)
Installation Succeeded
To activate this project's virtualenv, run pipenv shell.
Чтобы активировать виртуальную среду нашего проекта, надо ввести к-ду
pipenv shell

посмотрим установленные зависимости
pipenv graph
мы видим их (т.е. виртальная среда активирована)
openai==1.63.2
├── anyio 
│   ├── idna 
│   ├── sniffio 
│   └── typing_extensions 
├── distro 
├── httpx 
...

4. ___________________
Зайдем в файл Pipfile
там видим (в т.ч.)
[packages]
openai = "*" 

Зайдем в файл Pipfile.lock
Там мы увидим все зависимости, которые 
были видны при вводе к-ды

pipenv graph

5. !!!!!!!!!!!!
Так вот, 
теперь мы можем перенести 
весь проект (директорию python ?)
в другое место (на другой компьютер, например)
там выполнить к-ду 

pipenv install

и тогда, там сформируется такая же виртуальная
среда (с теми же зависмостями, которые
указаны в этих двух файлах - Pipfile, Pipfile.lock )

6. _____________________
Как найти свою виртуальную среду?

pipenv --venv

Зачем? 
Дело в том, что openai не находится в зависимостях, так как 
пайтон подключается внешний (см. справа внизу VC code, 
когда открыт файл пайтона, там у нас python 3.13.8 или
что-то похожее, а нам нужен пайтон из виртуального окружения,
иначе openai подсвечивается).
Нам надо сделать так, (после того, как путь найден).
Нажимаем на 3.13.8 справа внизу VC code, раскрываются 
переменные окружения, там мы вводим путь с уже исполняемым файлом пайтона
это

C:\Users\shaps\Desktop\Machine_learning\Bogdan\python\.venv\Scripts\python.exe
Enter

и openai уже не подсвечивается, т.е.
VC code корректно определил положение фиртуальной среды, 
исполняемый файл пайтона указан корректно
и внизу справа мы уже видим: 
Python 3.12.7 ('.venv')

7. ______________
Но code runner может не работать, для этого 
надо зайти в сетап и указать тот-же путь 
(с исполняемым файлом) там:

"code-runner.executorMap": {
        "javascript": "node",
        ...
        "python": "C:\\Users\\shaps\\Desktop\\Machine_learning\\Bogdan\\python\\.venv\\Scripts\\python.exe",
        ...
        }

8. _______________________
Минус такого варианта, что если мы создаем новую среду, мы должны выполнять пункты
6 и 7 заново.

9. Можно удалить все внутри папки .venv
 и пересоздать ее, т.е.
 перейти в папку 
 python
 и там ввести к-ду

 pipenv install
